# PRD：五子棋全端網站 (Python Flask)

## 專案背景
開發一個簡單的五子棋（Gomoku）網站，用於練習 **Flask 全端開發**與 **前後端整合**。玩家可以在瀏覽器上雙人對戰，人機對戰屬於後期擴充，不在本版本範疇。

***

## 核心需求
### 使用者故事
1. 玩家可以在瀏覽器中打開網站並進入遊戲界面。  
2. 玩家可選擇開始新遊戲，並透過同一螢幕兩人交替下棋。  
3. 畫面上呈現棋盤格線，棋子分別用黑白圓形表示。  
4. 遊戲會自動判斷是否有一方連成五子並彈出提示訊息。  
5. 玩家可選擇「重新開始」來重置棋盤。

***

## 功能需求
### 基礎功能
- 棋盤繪製：15x15 棋盤格。
- 雙人對戰：同一螢幕交替落子（無需註冊登入）。
- 勝負判斷：任一玩家連成五子（直、橫、斜線）即判勝。
- 重置遊戲：提供按鈕清空棋盤。

### 前端 (Flask Jinja2 + JavaScript)
- HTML5 + CSS 繪製棋盤 UI。  
- JavaScript 處理玩家落子點擊事件。  
- AJAX 請求與 Flask 互動（例如送出落子、查詢勝負）。  

### 後端 (Flask)
- 提供 API：  
  - `/`：首頁，顯示棋盤。  
  - `/move`：接收玩家落子資訊（座標與顏色），回傳當前棋盤狀態與是否勝負。  
  - `/reset`：重置遊戲狀態。  
- 後端維護棋盤狀態（陣列 15x15）。  
- 後端邏輯判斷五子連線。

***

## 非功能需求
- 前端簡單美觀，不需高級動畫效果。  
- 單一伺服器模式，無需使用資料庫（遊戲狀態存在記憶體）。  
- 網站需能在 Chrome 與 Edge 上正常運行。  
- 同時僅支援一局遊戲，不需多人連線或持久化。  

***

## 系統架構
### 架構設計
- **前端 Vue / React 非本版重點** → 使用純 HTML+CSS+JS + Jinja2。  
- **後端 Flask** → RESTful API 提供遊戲邏輯。  
- **狀態管理** → 使用 Python List/Dict 保留棋盤狀態於後端。  

### 資料流程
1. 玩家在棋盤上點選一個交叉點 (JS event listener)。  
2. 前端透過 AJAX 呼叫 `/move` API，傳遞 `(x, y, player)`。  
3. Flask 驗證該位置是否可落子，更新棋盤。  
4. Flask 執行勝負判斷，回傳 JSON：  
   ```json
   {
     "status": "ok",
     "winner": null,
     "board": [...]
   }
   ```
5. 前端更新畫面，若有勝利，顯示提示框。  

***

## 介面與 UX
- 初始進入：顯示 15x15 棋盤（空白）。  
- 點擊交叉點：落子並切換玩家顏色。  
- 提示區域：顯示當前回合的玩家。  
- 按鈕：  
  - 「重新開始遊戲」 → `/reset`。  
  - 「關於」 → 顯示本遊戲規則。  

***

## 未來擴充
- AI 對戰模式（透過簡單隨機或 Minimax 演算法）。  
- 玩家登入與分數保存。  
- WebSocket 即時多人對戰。
